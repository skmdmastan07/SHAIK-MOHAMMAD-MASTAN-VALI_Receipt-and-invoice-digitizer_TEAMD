DAY-4 — INFOSYS : SDLC NOTESSDLC — Software Development Life CycleA structured step-by-step process followed to design, build, test, deploy, and maintain software systems.It explains how software moves from idea to final usable product.SDLC RolesFunctional Consultant / Business AnalystUnderstands client requirements and business needs.Creates the Functional Specification Document which describes what the system should do.Developer / ProgrammerBuilds the actual software based on requirements.Creates the Technical Specification Document explaining how the system will be implemented.POC (Proof of Concept) — a small demo built to verify feasibility before full development.Implements logic in code level by writing programs.Performs Unit Testing to check each module works correctly.Tester / QA TeamTests the product in multiple scenarios to find defects.Creates Test Scripts — step-by-step testing instructions.Records Test Results to show pass or failure status.Main Server EnvironmentsDevelopment ServerUsed by developers to build and test features.Quality ServerUsed by QA team to validate functionality and stability.Production ServerLive environment used by real users.Administration TeamMaintains system infrastructure, permissions, configurations, and operational setup.Product / Project ManagementPlans, monitors, and controls the project progress.Ensures deadlines, scope, and resources are managed properly.Sales and MarketingSalesHandles client communication and closes deals.MarketingPromotes product through campaigns, ads, and feature awareness.Human ResourceManages hiring, employee relations, onboarding, and organizational policies.Documentation TeamRecords and maintains project information from beginning to end.Helps future reference, maintenance, and knowledge transfer.Support TeamHandles issues or bugs after deployment.Major problems are escalated back to the development team.Development ModelsWaterfall ModelA sequential process where each phase must finish before the next begins.Agile MethodologiesAn iterative approach where development happens in small cycles with continuous feedback.Agile RolesProduct OwnerDefines requirements and prioritizes features.Scrum MasterEnsures team follows Agile practices and removes obstacles.Development TeamDesigns, develops, tests, and deploys the product.Agile FlowDesign → Develop → Test → Deploy in repeated cycles.
